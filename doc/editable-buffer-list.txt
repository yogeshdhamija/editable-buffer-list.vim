*editable-buffer-list.txt*

Author: Yogesh Dhamija <yogeshdhamija@outlook.com>

This plugin provides |:BufferList|, which turns the current window into the
buffer list -- as a plain, editable buffer. It is to buffers roughly what
oil.nvim is to files: the list is just text, and editing the text edits
reality.

The text is |:ls| output, verbatim -- same columns, same flags, same
alignment. It is produced by running |:ls| in the list's own window, which is
what makes the markers read correctly: the list buffer is 'nobuflisted', so it
never appears in its own output and no line is ever marked `%`. The `#` sits on
the buffer the window was showing before |:BufferList| replaced it, and stays
there no matter what other windows do.

Deleting a line unloads (|:bdelete|) its buffer, immediately. Because the
list is a real modifiable buffer, anything that removes text works: `dd`,
`5dd`, `d2j`, `dG`, `Vd`, `:g/foo/d`. Nothing is remapped for this.

`u` brings an unloaded buffer back. This falls out of how the sync works:
undo restores the line, and since |:bdelete| only unlists a buffer rather
than destroying it, the restored line relists it. `CTRL-R` redoes.

<CR> opens the buffer under the cursor in this window. The alternate file is
preserved (|keepalt|), so CTRL-^ afterwards goes to the buffer the window
showed before the list -- as if the list was never there. That is also the
way to leave without picking anything.

Buffers with unsaved changes refuse to unload: the line comes back and the
|E89| message is shown. Save the buffer first, or use `:bdelete!` yourself if
you mean to discard the changes. The same happens for anything else
|:bdelete| refuses, such as a terminal with a running job.

Only the leading buffer number on each line is ever read back; the rest of
the line is redrawn from the real buffer list after every change. So mangling
a line's text cannot corrupt anything -- the line is simply redrawn. A
side effect: writing a bare buffer number on its own line relists that
buffer, if it exists.

Navigating away from the list leaves it in the |jumplist|, so CTRL-O comes
back to it. It is redrawn on arrival, so a list returned to this way shows the
buffer list as it is now, not as it was when you left.

The list buffer itself is 'buftype' nofile and 'bufhidden' hide: it never
needs saving (changes apply as you make them) and never touches the buffer
list it is displaying. It is kept rather than wiped when hidden, because a
wiped buffer has its |jumplist| entries dropped along with it. Abandoned lists
are reused by the next |:BufferList| rather than piling up; a list still
visible in some window is never taken over, so each window keeps its own.
